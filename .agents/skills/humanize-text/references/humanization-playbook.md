# Humanization Playbook

Use these tactics to make prose sound more natural without turning it into performance.

## Core principle

Human-sounding prose is not slangier prose.
It is prose with clearer intent, stronger information density, more natural sentence movement, and better local specificity.

## Tactics

### 1. Lead with the real point
Replace setup language with the claim, action, or observation the reader actually needs.

### 2. Trade abstraction for concrete language
Prefer:
- specific actors over passive structures
- direct verbs over noun-heavy phrasing
- visible outcomes over generic benefits

### 3. Remove filler first
Delete words that only signal emphasis or transition unless they carry real contrast.

Common cuts:
- really
- very
- basically
- essentially
- in order to
- it is important to note
- in conclusion

### 4. Break predictable rhythm
If multiple sentences have the same shape, vary one of these:
- opening structure
- sentence length
- clause depth
- emphasis position

Do not randomize. Vary only where monotony is audible.

### 5. Collapse duplicated ideas
When two sentences do the work of one, merge or delete.

### 6. Keep one main move per sentence
When a sentence tries to define, qualify, summarize, and persuade all at once, split or simplify it.

### 7. Name the actor
If the reader cannot tell who is doing what, rewrite in active voice unless passive voice is necessary.

### 8. Ground the copy in local context
Use only details supported by the surrounding file or provided text.
Possible grounding points:
- product or feature name
- user action
- actual constraint
- concrete result
- target audience

### 9. Preserve honest uncertainty
Do not invent confidence. If the source is tentative, keep it tentative.

### 10. Stop before overcorrection
Do not add fake personality markers such as:
- casual slang
- typos
- rhetorical theatrics
- invented anecdotes
- emoji-like phrasing

## Rewrite priorities

Apply these in order:
1. clarity
2. specificity
3. concision
4. rhythm
5. tone smoothing

## Safe rewrite patterns

### Before
"In today's fast-paced digital environment, it is important for businesses to leverage innovative solutions in order to enhance user engagement."

### After
"Businesses need clearer, more useful product experiences if they want users to stay engaged."

### Before
"Additionally, this feature provides a streamlined and efficient way for users to complete tasks in a seamless manner."

### After
"This feature helps users finish the task with fewer steps."

## Final check

Before finishing, ask:
- Did the rewrite keep the original claim intact?
- Did it become more specific where context allowed?
- Did I remove padding instead of merely paraphrasing it?
- Did I avoid adding fake humanity?
